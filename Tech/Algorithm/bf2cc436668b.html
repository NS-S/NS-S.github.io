数据结构学习笔记#2 栈与队列

之所以要引入这样的数据结构，是为了简化问题，将思考范围减小，更加聚焦于问题核心。

栈 Stack

LIFO，Last In First Out

实现方式

由于实现方式没有特别之处，所以不再赘述。
栈的顺序存储和链式存储，主要区别有二：

  • 顺序存储有数量限制，而链式存储没有
  • 链式存储有空间时间浪费（申请空间所需时间），而顺序存储没有

依据元素数量可控/不确定，来确定实现方式。

顺序存储中，两栈共享空间

如果有两个栈有相反的空间需求，那么可以把他们用同一个数组表示：一个从前往后增长，一个从后往前增长。
栈数据结构需要保存两个子栈的顶指针（下标），而各种栈操作则需要加一个栈的编号的参数（1还是2）。

应用

实现函数调用

系统用栈来保存函数上下文，实现函数之间的调用关系(后进的函数,最先被弹出)。

四则运算表达式求值 - 逆波兰表示法

在逆波兰表示法中，无论是“把中缀表达式转化成后缀表达式”，还是“计算后缀表达式”，都需要用栈作为工具。
我写了一个简单的表达式转换程序。（仅支持个位数加减法）

队列 Queue

FIFO, First In First Out

实现方式

队列长度确定用顺序存储（循环队列），不确定用链式存储。

顺序存储

如果用普通的顺序存储，可以发现要么出队列的复杂度很高（前面走一个，后面全要挪），要么会出现“假溢出”现象（前面空的，后面又不挪）。

所以有了循环队列，用两个“指针”front和rear，分别指向首元素和末元素的下一位。（其实只是下标而已）

判断队列为空，就用front==rear；
判断队列已满，有两种实现方法：

  1. 采用一个额外的flag变量区分空与满。
  2. 定义满为还差一个就满了 (rear + 1) % MAXSIZE == front

计算队列长度，用 (rear - front + MAXSIZE) % MAXSIZE

（可以发现循环队列很多操作都会%MAXSIZE）

链式存储

为了操作方便，使用一个队头指针和一个队尾指针，分别指向没用的头节点和尾节点。
就是非常普通的链表操作所以不赘述了。

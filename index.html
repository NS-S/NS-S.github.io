一星七#1

本周干的事情还挺多的。（这下真考验记忆力了）

首先是礼拜二叫了朋友来家里玩，玩了很多东西，包括展示我 精湛 的非洲鼓技巧（指没学过非洲鼓）、欣赏优美钢琴、视奏考试、协调能力考验、玄学网球等。晚上去黄渡理工大学的嘉实广场逛了一圈，吃了顿十分充实的萨莉亚，饭后散步N公里散到地铁站，然后再晚间运动骑自行车回家。
和这两位朋友聚还是非常开心滴，不像有人从生日两天前开始人间蒸发，生日当天还装死的。

然后这周最头疼的事情不是ROP题，是周六出了高考成绩，周日要帮学校招生，打二十几个学弟学妹电话一个个问高考多少分……要是学弟学妹们高考成绩不错也挺好，我可以帮着参谋参谋。关键是我打到的最高分也才刚满去年的最低分数线啊！！
感觉我不是在打电话，我是在寄刀片有么有啊！
所以打了五个电话之后我就学聪明全打母亲的电话了，一方面家里学习一般都是妈妈管，另一方面家长比学生好交流，主要是家长不会和学生本人一样直接破防的。（指问了成绩之后沉默十几秒然后挂我电话）

还是说说学习方面吧！

区块链下周就考试了，所以我从头把PPT都看了并且做了点Anki卡片。有些东西再看还是很难理解，比如环签名、群签名，又比如比特币的P2SH。课程PPT质量不好，网上又懒得找好资料看，就暂且摆烂吧……（用到了再学）
但是有一点收获，我有点看懂以太坊黄皮书里的以太坊架构了！不过我显然懒得看黄皮书原文，我指的是Lee Thomas制作的以太坊机制图。如果让我选一张我现在看过最精妙复杂的图片媒介，我一定会选这张。这张图完美地诠释了到底什么叫做科技黄图。
如果我以后要制图，一定以这张图为终极参考目标，学习其中的各种美化技巧！但是我好像没什么要制图的场景哦。

这周打算开始自学数据结构了，开学前想把《大话数据结构》看完，并且刷一些题（这是有时间的吗）。这周看了四章：数据结构算法简介、线性表、栈和队列。这本书我觉得小学生都看得懂，有大量类比，并且内容也很浅。

最后是PWN相关。本周把pwn.college的babyrop从8.0一直推到15.0， 打通关了！！！
自我感觉学到了很多技巧方法，都记录在我的writeup里了，可惜这门课练习的writeup不能公开发表到互联网。下周准备给博客加一个加密的插件，把我的writeup加密之后放上来，输入密码才能看。（师傅告诉我的小操作）

但是我也越发隐隐约约地觉得：我还有好多好多东西都没学啊！真希望自己能在pwn的世界里学到更多技巧，在赛博空间里横行霸道、横冲直撞、打来打去、盗亦有道（指东从.text拿一个gadget，西从libc拿一个gadget）……

结语

综上，本人记忆力还是不错的！
那为什么我求导公式都记不全啊（

数据结构学习笔记#0

读《大话数据结构》的读书笔记，读了更专业的书以后会继续更新笔记。

数据结构相关概念介绍

计算机，听名字就知道一开始被发明，是用来完成计算任务的——比如各种物理计算等等。
在各种物理计算之中，都离不开物理模型的存在。有了物理模型，我们得以把复杂的现实简化，提取其中的关键之处用于计算。
而在用计算机处理各种问题的时候，我们也需要将复杂的现实抽象成数据，能够被计算机处理的数据。（这里就不仅仅局限于早期的数值计算了，而是广义的各种问题）
这就引出了数据相关的一系列定义……

数据

数据： 描述客观事物的符号，是计算机中可以操作的对象，是能被计算机识别，并输入给计算机处理的符号合集。
数据不仅包括int、float等我们在编程的时候用到的基础类型，还包括声音、图像、视频等非数值类型，这些类型会以二进制数据的形式存储在计算机中，所以也是数据。

接下来借一段C语言，解释一下数据元素、数据项和数据对象。

数据元素指Book，是组成数据的基本单位；
数据项指ID、name和author，是数据元素的属性，数据的最小单位；
数据对象指myBookShelf，是我书架的数据，一般就简称数据了。

数据结构

数据结构： 相互之间存在一种或多种特定关系的数据元素的集合。
数据结构有两种，一种称为逻辑结构，一种称为物理结构。
逻辑结构是抽象的数据结构，通过物理结构来实现。

逻辑结构

  1. 集合结构 各个数据元素平等，共同属性是同属一个集合，别的没了。
  2. 线性结构 各个数据元素可以连成一条线（给定顺序后可以说是一对一）。
  3. 树形结构 各个数据元素间存在一对多的关系。
  4. 图形结构 各个数据元素间存在多对多的关系。

物理结构

  1. 顺序存储结构 以连续的地址存储数据。
  2. 链式存储结构 以任意方式存储数据，再通过指针将其串联在一起。

数据类型

数据类型： 一组性质相同的值的集合及在此集合上的一些操作的总称。
由于我们的数据类型百花齐放，在不同程序里的数据类型都不同（可用操作不同等等），所以我们又设计了一个概念：
抽象数据类型（Abstract Data Type，ADT）： 一个数学模型及定义在该模型上的一组操作。
所以之后我们都会研究各种ADT的性质，在实际使用时借助ADT的概念，因地制宜实现出适合的数据结构。

算法概念介绍

算法和数据结构是离不开的，我随便举个例子：
数组和链表先比，最大的优点便是可以随机读取。那么一个需要随机读取的算法在数组上复杂度就是O(1)，而在链表上就是O(n)。

简单地说，我们设计一个好的算法要往以下几方面努力：

  1. 正确性&健壮性&鲁棒性
  2. 时间复杂度低，空间复杂度低
  3. 可读性&可理解性

那么问题来了：如何检验一个算法的这三种能力（尤其是前二种）？
在二进制程序分析中有静态分析和动态分析的概念，而对于算法也有事前分析估算方法和事后统计方法。

由于后者常常受到软硬件各种限制，我们最常用的方法是事前分析估算方法。对于时间复杂度的事前分析估算方法尤为重要，但我在这里不详述了，因为严谨地讲比较烦，翻翻数学分析书就知道了。（2333）
总之，时间复杂度上，有下列关系（注意：是在一定大小的N之后，才满足下列大小关系）：
O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)

空间复杂度也使用大O记法，直接处理数据连存都不存就是O(1)，全部存下来就是O(n)。空间和时间复杂度可以相互转换，类似于预录制过场动画和即时演算画面的差别。（？）
另外，我们一般说一个算法的复杂度，都是指时间复杂度，因为这个日常影响更大一些。

最后还是想吐槽一下，《大话数据结构》真的好懂到休闲读物的程度。
